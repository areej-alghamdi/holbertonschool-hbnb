HBnB Evolution - Part 1: Technical Documentation
Welcome to the Technical Documentation phase of the HBnB Evolution project. This directory contains the complete architectural blueprints, entity models, and system sequence flows designed for the platform before the implementation phase.

📋 Objectives & Tasks Covered
In this initial part, we focused on laying down the foundational design from three major perspectives:
High-Level System Architecture: Implementing a layered configuration.
Business Logic Layer Modeling: Mapping domain objects, attributes, and actions.
API Interaction Flows: Tracing data communication paths at runtime.

---

📂 Directory Map & Deliverables
To keep the documentation clean and scannable, each core requirement has been separated into its own specialized file:

[Full Compilation Document](./Documentation.md): The comprehensive technical document that combines all notes, business rules, and references.
[Task 0: Package Diagram](./package_diagram.md): Details the three-layer architecture (Presentation, Business Logic, and Persistence) and explains the Facade Pattern communication.
[Task 1: Detailed Class Diagram](./class_diagram.md): Depicts the core domain models (User, Place, Review, Amenity), their methods, attributes, and relationships.
[Task 2: Sequence Diagrams](./sequence_diagram.md): Visualizes the step-by-step API calls for User Registration, Place Creation, Review Submission, and Fetching Places.
